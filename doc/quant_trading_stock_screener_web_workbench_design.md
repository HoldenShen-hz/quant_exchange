## 1. 文档目标

本文档用于补充说明 `quant_exchange` 平台中的股票筛选网页工作台，重点解决以下需求：

- 在统一界面浏览 A 股、港股、美股股票目录
- 通过板块、行业、概念和财务指标进行条件选股
- 通过 F10 简况进行文本筛选
- 对单只股票进行财务分析
- 对单只股票保存和查询历史财务数据
- 对单只股票展示历史 K 线图
- 对单只股票保存和查询分钟级交易数据
- 提供独立的加密货币研究页签
- 展示加密货币市场概览、币种详情和历史 K 线图
- 在网页中展示后台实时市场快照
- 提供 watchlist、筛股预设和市场脉搏卡片
- 提供历史数据下载中心，展示任务状态和控制按钮
- 提供网页可见的模拟交易页签与纸面账户操作面板
- 提供面向金融与量化新手的学习中心
- 将两只股票进行指标与摘要对比
- 记录网页关键操作并展示最近操作记录
- 再次打开工作台时自动恢复上次保存的筛股和对比状态
- 使用接近主流交易 App 的分区式布局，拆分为首页、学习、自选、选股、F10、图表、加密、对比、模拟、数据和动态，不将所有功能堆叠在同一页
- 在“动态”页中提供参考 `FMZ` 的策略机器人中心
- 支持从当前股票快速创建机器人，并进行启动、暂停、停止、同步和人工清仓
- 提供机器人通知中心，展示机器人生命周期与交互消息
- 首页需要提供快捷入口、市场脉搏、研究焦点和操作路径提示
- 桌面端使用顶部主导航，小屏设备提供底部导航
- 数据页需要提供历史下载汇总条和各市场下载任务卡片

## 2. 设计范围

当前模块定位为“股票研究入口”和“候选池构建工具”，不直接承担实盘交易终端职责。

支持范围：

- 股票目录浏览
- 多条件筛股
- F10 简况卡片
- 财务分析卡片
- 历史财务快照
- 历史 K 线图
- 加密货币市场页
- 分钟级交易数据
- 后台实时市场快照
- 历史数据下载中心
- 纸面模拟交易工作台
- 学习中心
- 策略机器人中心
- 机器人通知中心
- 观察列表
- 筛股预设
- 市场脉搏卡片
- 双股票对比
- JSON API
- 网页工作台
- App 风格工作台导航
- 用户登录与用户空间
- 工作台状态持久化
- 操作日志留存与查询

不在当前首版范围内：

- 实时逐笔行情图表
- 多指标叠加分析图层
- 全市场实时主数据自动同步

## 3A. 加密货币研究页

系统应在网页工作台中提供独立的“加密”页签，采用与股票页一致的 App 风格布局，但数据口径独立。

至少包括：

- 加密货币市场概览卡片
- 币种列表
- 币种详情卡片
- 历史 K 线图
- 按用户保存的当前币种和图表状态

当前实现基于内置 `SIM_CRYPTO` 适配器，已支持 `BTCUSDT / ETHUSDT / SOLUSDT / BNBUSDT / DOGEUSDT` 的页面展示、详情查看、历史 K 线接口和用户状态恢复。

## 3. 功能定义

### 3.1 股票目录

系统需要维护统一股票主数据目录，至少覆盖：

- 代码
- 名称
- 市场区域
- 交易所
- 板块
- 行业
- 概念标签
- 基础财务指标
- F10 简况字段

当前实现已经支持从官方主数据源导入 A 股、港股、美股股票目录，并持久化到本地数据库；页面筛选基于全市场股票池执行，但默认只渲染前一批结果，以避免在几千到上万只股票时拖慢网页响应。

### 3.2 条件选股

系统应支持以下筛选维度：

- 市场：A 股 / 港股 / 美股
- 交易所：如 `SSE / SZSE / HKEX / NYSE / NASDAQ`
- 板块
- 行业
- 概念标签
- 名称 / 代码 / 关键字
- `PE / PB / ROE / 营收增速 / 净利润增速 / 股息率 / 负债率 / 市值`

### 3.3 F10 简况筛选

系统应支持对以下字段做文本匹配：

- 公司简况
- 主营业务
- 产品与服务
- 竞争优势
- 风险提示

F10 筛选结果必须能解释用户为何命中该股票。

### 3.4 双股票对比

用户可以在结果列表中选择任意两只股票进行并排对比，对比内容至少包括：

- 估值
- 盈利能力
- 成长性
- 分红
- 负债
- F10 摘要

### 3.5 财务分析

系统应支持对单只股票输出结构化财务分析结果，至少包括：

- 综合财务评分
- 估值评分
- 盈利能力评分
- 成长性评分
- 现金流评分
- 偿债能力评分
- 财务分析摘要
- 财务亮点
- 财务关注点

### 3.6 历史财务数据

系统应支持为每只股票保存历史财务快照，至少包括：

- 报告期
- 营业收入
- 净利润
- 经营现金流
- 自由现金流
- 毛利率
- 净利率
- ROE
- 资产负债率
- 流动比率与速动比率

系统应支持按股票和报告期查询这些历史财务数据，为财务趋势分析、因子计算和基本面筛选提供数据基础。

### 3.7 历史 K 线图

系统应支持对单只股票展示最近一段历史日线 K 线图，至少包括：

- `open / high / low / close / volume`
- 最新收盘价
- 最近区间涨跌幅
- 最近区间高点和低点

实现要求：

- 对已下载本地历史文件的 A 股，优先使用真实历史数据
- 对没有本地历史文件的股票，允许生成稳定可复现的演示历史序列
- 页面在切换股票时，应自动刷新对应 K 线图

### 3.8 分钟级交易数据

系统应支持为每只股票保存分钟级交易数据，至少包括：

- 分钟时间
- `open / high / low / close`
- 成交量
- 成交额
- 市场区域与交易所

首版实现应支持：

- 后台实时行情刷新时持续写入当前分钟数据
- 在没有真实分钟行情源时，可生成稳定可复现的演示分钟线并落库
- 提供按股票查询最近 `N` 根分钟线的接口

### 3.9 观察列表与筛股预设

系统应支持以下增强交互：

- 将股票加入观察列表并快速再次打开
- 保存观察列表到工作台状态
- 提供常用筛股预设，例如质量、成长、股息和价值风格
- 提供市场脉搏卡片，快速显示质量龙头、成长龙头、股息领先和价值观察

### 3.10 后台实时市场快照

系统应支持由后台常驻服务持续刷新当前股票池和全市场快照，并为网页提供：

- 最新价
- 涨跌额和涨跌幅
- 成交量和成交额
- 市场状态，如 `OPEN / PRE / POST / CLOSED`
- 全市场摘要，如涨跌家数、最强股票、最弱股票和最活跃股票
- 根据市场状态返回推荐轮询频率，使网页在 `OPEN` 时段高频刷新，在 `CLOSED` 时段降频检查

首版实现采用后台线程 + JSON 轮询方式，优先保证稳定性和简洁部署，不依赖长连接基础设施。

### 3.11 工作台状态恢复

系统应支持保存以下最近一次工作状态：

- 当前筛股条件
- 当前查看股票
- 当前左侧和右侧对比股票
- 当前观察列表
- 当前图表区间和图表模式
- 当前筛股预设
- 当前学习章节

再次打开网页时，系统应自动加载最近一次保存状态，减少重复筛选成本。

系统进入多用户模式后，应支持按“用户账号优先、浏览器 `client_id` 兜底”的方式隔离状态。

### 3.12 网页操作记录

系统应记录至少以下关键操作：

- 页面打开
- 执行筛股
- 重置筛股
- 查看股票详情
- 选择左侧或右侧对比股票
- 观察列表增删
- 预设筛股切换
- 图表模式和区间切换
- 打开学习课程
- 提交学习测验

操作记录应支持按时间倒序返回，供页面显示最近操作列表。

### 3.13 策略机器人中心

系统应在“动态”页提供参考 `FMZ` 风格的机器人管理区，至少包括：

- 策略模板列表
- 基于当前选中股票快速创建机器人
- 机器人运行状态：`draft / running / paused / stopped`
- 运行时指标：最新价、价格变化、目标仓位、信号原因、最近心跳
- 常用交互命令：
  - 启动
  - 暂停
  - 停止
  - 同步信号
  - 更新参数
  - 人工清仓

首版机器人中心以研究 / 模拟用途为主，不直接承担真实交易所托管执行。

### 3.14 机器人通知中心

系统应在“动态”页展示机器人通知流，至少包括：

- 创建成功通知
- 启动 / 暂停 / 停止通知
- 参数更新通知
- 人工清仓通知
- 最近通知的时间倒序展示

通知中心应与机器人交互保持联动，操作成功后页面可自动刷新并显示新通知。

### 3.15 模拟交易页签

系统应在“模拟”页提供面向研究和人工演练的纸面交易面板，至少包括：

- 模拟账户总览：
  - 总权益
  - 可用现金
  - 持仓市值
  - 未实现盈亏
  - 已实现盈亏
- 订单录入：
  - 当前标的提示
  - 买入 / 卖出方向切换
  - 市价 / 限价切换
  - 数量输入
  - 限价输入
- 账户联动信息：
  - 当前持仓列表
  - 最近订单
  - 最近成交
  - 当前标的的策略目标仓位偏差
- 常用操作：
  - 刷新账户
  - 提交模拟订单
  - 撤销未成交订单
  - 重置模拟账户

首版实现以股票纸面账户为主，优先支持手工研究交易和链路验证，不替代真实实盘终端。

### 3.16 新手学习中心

系统应在“学习”页提供适合金融和量化新手的课程入口，至少包括：

- 知识库：
  - 覆盖宏观经济、财经周期、货币、利率、汇率、市场结构、股票、财报、估值、债券、期货、期权、区块链、交易执行、量化研究、风控、银行、保险、监管与行为金融
  - 支持按主题卡片展示知识点
  - 支持按术语、主题和关键字进行知识库检索
- 学习计划：
  - 按阶段拆解学习目标
  - 为每阶段给出建议时长和产出
- 学习内容：
  - 按课程展示目标、重点内容和课后练习
  - 支持从课程目录切换当前章节
- 学习检验：
  - 提供章节相关测验题
  - 支持自动评分
  - 支持错题解释与建议复习章节

同时，系统应支持单独通过 `/learn` 访问学习中心，并与工作台状态保存联动。

### 3.17 多用户与用户空间

系统应支持网页端多用户使用，至少包括：

- 用户注册
- 用户登录 / 退出
- 当前用户识别
- 用户级工作台状态保存
- 用户级学习进度与测验历史保存
- 用户级操作日志查询
- 用户级纸面模拟账户隔离

当用户未登录时，系统仍允许按浏览器 `client_id` 保存临时工作状态；一旦登录，应优先切换到该用户独立空间。
